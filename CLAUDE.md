# CLAUDE.md

## 项目概览

本仓库是 [Novaix](https://github.com/huohuastudio/novaix) 的对外文档站，基于 VitePress 构建。Novaix 文档位于 `docs/novaix/` 目录下。

## 文档写作规范

### 语气与称谓

- 全篇使用**中文**，技术术语保留英文原文（如 OVN、CIDR、cloud-init）
- 统一使用「您」称呼读者，不用「你」
- 语气**平实、务实、直接**，像一位有经验的同事在写操作手册——讲清楚是什么、怎么做、要注意什么，不多说废话
- 不使用营销化表述（「强大的」「极致的」「一站式」）、不夸张、不渲染情绪
- 不使用 emoji 作为评级或装饰

### 页面结构

- 每个页面以 `# 标题 {#anchor}` 开头，紧接一两句话概述该功能是什么、解决什么问题
- 用 `##` 和 `###` 组织小节，每个标题都带 `{#anchor}` 便于链接定位
- 开篇概述后直接进入操作步骤或具体说明，不需要「本文将介绍……」之类的导读段落

### 内容组织

- **表格**：用于字段说明、状态说明、对比说明，列名通常是「字段 | 说明」或「项目 | 要求」
- **列表项**：用 `**加粗名称**` + 冒号或破折号 + 简短说明，一行讲清一件事
- **代码块**：命令行示例用 `bash` 标注语言，关键步骤加行内注释
- **流程图**：复杂流程使用 mermaid 图辅助说明
- **交叉引用**：用 `[文字](./page)` 或 `[文字](./page#anchor)` 引用相关页面，不要写死 URL

### 提示框使用

- `:::tip`：补充建议、最佳实践、省心的做法
- `:::warning`：注意事项、容易踩的坑、前置条件
- `:::danger`：不可逆操作、严重后果、数据安全
- 提示框内文字简洁，直接说结论和原因，不铺垫

### 措辞习惯

- 说「建议」而非「强烈推荐」，说「请确保」而非「务必请您一定确保」
- 因果关系用「否则……」「如果……会导致……」直接说后果
- 常见搭配：「详见[xxx](./xxx)」「参考[xxx](./xxx#anchor)章节」「您可以在管理面板的「xxx」页面……」

### 禁止出现的内容

- 所有用户可见的文案中**禁止出现 "Incus" 字样**，使用「运行环境」「节点」「管理服务」等通用表述替代
- 不使用「Card 套 Card」等内部开发术语
