# 邮件配置 {#mail}

Novaix 使用邮件发送通知，包括工单回复通知、密码重置验证码、到期提醒等。邮件服务采用插件化设计，支持多种发送渠道。

## 支持的渠道 {#channels}

| 渠道 | 说明 |
|------|------|
| SMTP | 传统邮件发送协议，适用于任意邮件服务商 |
| Mailgun | Mailgun 邮件发送 API，适合大量发送 |
| Resend | Resend 邮件发送 API，开发者友好 |

在管理面板的「系统设置」→「邮件设置」中选择并配置邮件渠道。每个渠道的配置项会以动态表单展示，配置后立即生效，无需重启程序。

## SMTP {#smtp}

| 字段 | 说明 |
|------|------|
| SMTP 服务器 | 邮件服务器地址，如 `smtp.gmail.com` |
| 端口 | SMTP 端口，通常为 465（SSL）或 587（TLS） |
| 加密方式 | SSL、TLS 或无加密 |
| 用户名 | SMTP 登录用户名 |
| 密码 | SMTP 登录密码或应用专用密码 |
| 发件人名称 | 邮件中显示的发件人名称 |
| 发件人地址 | 邮件中显示的发件人邮箱 |

::: tip
常用邮件服务商的 SMTP 配置：
- **Gmail**：`smtp.gmail.com`，端口 587，需要使用[应用专用密码](https://support.google.com/accounts/answer/185833)
- **阿里云企业邮箱**：`smtp.qiye.aliyun.com`，端口 465
- **腾讯企业邮箱**：`smtp.exmail.qq.com`，端口 465
:::

## Mailgun {#mailgun}

| 字段 | 说明 |
|------|------|
| API Key | Mailgun API 密钥 |
| Domain | 已验证的发送域名 |
| Region | API 区域，`us`（美国，默认）或 `eu`（欧盟） |
| 发件人名称 | 邮件中显示的发件人名称 |
| 发件人地址 | 邮件中显示的发件人邮箱 |

## Resend {#resend}

| 字段 | 说明 |
|------|------|
| API Key | Resend API 密钥（以 `re_` 开头） |
| 发件人名称 | 邮件中显示的发件人名称 |
| 发件人地址 | 邮件中显示的发件人邮箱 |

::: tip
Resend 提供免费额度，适合小规模使用。注册后在 [Resend Dashboard](https://resend.com) 中获取 API Key 即可。
:::

## 测试 {#test}

配置完成后，建议点击「测试」按钮发送一封测试邮件，确认配置正确。测试邮件会发送到您指定的邮箱地址。

::: warning
未配置邮件渠道时，所有需要发送邮件的功能（验证码、密码重置、工单通知、到期提醒等）都会**静默跳过**，不会报错也不会有任何提示。如果您发现用户收不到验证码或通知邮件，请先检查邮件渠道是否已正确配置。
:::

## 邮件模板 {#templates}

Novaix 内置了多种邮件模板，您可以在「系统设置」→「邮件模板」中自定义模板内容。系统会自动替换模板中的变量（如用户名、验证码、实例信息等）。

## 邮件接收（Inbound） {#inbound}

Novaix 支持通过接收邮件来自动回复工单。用户直接回复工单通知邮件，邮件内容会自动添加到对应工单中。

该功能需要配置邮件接收的 Webhook 地址：

```
https://您的域名/api/callbacks/mail/inbound
```

具体的配置方式取决于您使用的邮件服务商，通常在邮件服务商的「Inbound Parse」或「Webhook」设置中配置。
