# 用户管理 {#user}

Novaix 提供完善的用户管理功能，支持用户注册、邮箱验证、密码重置、双因素认证等。

## 用户列表 {#user-list}

管理员可以在管理面板的「用户管理」中查看所有用户，支持按用户名、邮箱搜索，支持导出用户列表到 Excel。点击用户名可进入用户详情页。

## 用户详情 {#user-detail}

用户详情页通过 8 个 Tab 展示用户的全维度数据：

| Tab | 说明 |
|-----|------|
| 概览 | 用户基本信息、余额、实例数量、订单数量等汇总数据 |
| 个人资料 | 查看和编辑用户的基本信息、代理设置、安全设置 |
| 实例 | 该用户名下的所有实例列表 |
| 订单 | 该用户的所有订单记录 |
| 支付记录 | 该用户的所有支付流水 |
| 交易记录 | 该用户的余额变动明细 |
| 工单 | 该用户提交的所有工单 |
| 操作日志 | 该用户的操作行为记录 |

在详情页顶部，管理员可以快速执行以下操作：

- **一键登录**：以该用户身份打开前台面板（不影响管理员当前会话），方便排查用户问题
- **充值 / 调账**：直接为用户充值或调整余额
- **编辑**：修改用户的基本信息和角色
- **删除**：删除该用户账号

## 创建用户 {#create-user}

管理员可以手动创建用户，也可以开放注册功能让用户自行注册。注册时支持邮箱验证码或手机短信验证码校验。

::: warning
如果您开放了用户注册功能，强烈建议同时开启邮箱验证。未开启邮箱验证的情况下，任何人都可以用任意邮箱注册账号，可能会带来恶意注册的风险。邮箱验证功能依赖 [SMTP 邮件服务](./mail)，请确保已配置。
:::

## 用户功能 {#user-features}

### 账户安全 {#account-security}

- **邮箱验证**：注册和密码重置支持邮箱验证码
- **手机号验证**：支持手机号注册、登录和密码重置（需要先配置[短信服务](./sms)）
- **双因素认证**（TOTP）：管理员默认启用，普通用户可自行开启
- **登录保护**：连续登录失败后会临时锁定账号，防止暴力破解
- **API 密钥**：用户可以生成 API 密钥用于接口调用，支持细粒度的权限控制

::: tip
管理员账号的双因素认证（TOTP）默认强制启用，首次登录后必须绑定 TOTP 才能正常使用管理面板。建议使用 Google Authenticator、Microsoft Authenticator 或其他兼容的 TOTP 应用。请务必妥善保管 TOTP 密钥的恢复码，如果丢失将无法登录管理面板。
:::

### 个人资料 {#profile}

用户可以在用户面板中管理自己的个人信息，包括：

- 修改密码
- 管理邮箱（添加、删除、设为主邮箱）
- 绑定 / 换绑手机号
- 绑定 / 解绑双因素认证
- 管理 SSH 密钥（用于创建实例时自动注入，方便免密码 SSH 登录）
- 管理 API 密钥
- 查看账户余额和充值

### 多邮箱管理 {#multiple-emails}

用户可以绑定多个邮箱地址，任意已验证的邮箱都可以用于登录和找回密码。

- **添加邮箱**：输入新邮箱地址，系统发送验证码到该邮箱，验证通过后完成绑定
- **设为主邮箱**：切换主邮箱，主邮箱用于接收系统通知（订单提醒、到期通知等）
- **删除邮箱**：移除不再使用的邮箱，但不能删除主邮箱（需先切换到其他邮箱）

如果用户开启了双因素认证，添加、删除和切换主邮箱操作都需要输入 TOTP 验证码。

### 手机号管理 {#phone-management}

如果管理员配置了[短信服务](./sms)，用户可以绑定手机号：

- 绑定后可以使用手机号 + 短信验证码登录和找回密码
- 换绑手机号时需要通过当前手机号的短信验证

### 钱包 {#wallet}

每个用户都有一个账户余额（钱包），可以用于支付订单。管理员可以手动调整用户余额。

::: tip
用户可以通过配置好的支付渠道向钱包充值。充值后的余额可以用于支付任意订单。退款也会退回到钱包余额中，而不是原支付渠道。
:::

## 用户角色 {#roles}

管理员可以修改用户的角色：

- **普通用户**：默认角色，可以购买和管理自己的实例
- **代理商**：拥有推广链接和佣金体系，详见[代理系统](./agent)
- **管理员**：拥有全部管理权限

::: danger
请谨慎授予管理员角色。管理员拥有对所有用户数据、所有节点、所有实例的完全控制权，包括打开任意实例的终端和控制台。建议仅为必要的人员分配管理员角色。
:::

## 删除用户 {#delete-user}

::: warning
删除用户前请确认该用户名下没有活跃的实例和未完成的订单。如果用户名下还有运行中的实例，建议先处理这些实例（删除或转移），再删除用户账号。直接删除有实例的用户可能导致实例变成无主状态。
:::
