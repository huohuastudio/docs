# 短信服务 {#sms}

Novaix 支持通过短信发送验证码，用于用户注册、登录和密码重置。短信服务采用插件化设计，支持多种短信渠道。

## 支持的渠道 {#channels}

| 渠道 | 说明 |
|------|------|
| 阿里云短信 | 阿里云 SMS 服务，国内常用 |
| 腾讯云短信 | 腾讯云 SMS 服务 |
| 通用 HTTP | 适配任意支持 HTTP 接口的短信平台 |
| 短信宝 | 低价短信平台，无需企业认证（官方插件） |

每个渠道的配置项会以动态表单展示，填写并保存后即可使用。配置完成后，建议点击「测试」按钮发送一条测试短信，确认渠道配置正确。

## 阿里云短信 {#aliyun}

| 字段 | 说明 |
|------|------|
| AccessKey ID | 阿里云 RAM 用户的 AccessKey |
| AccessKey Secret | 对应的 Secret |
| 短信签名 | 已审核通过的签名内容 |
| 模板 CODE | 已审核通过的验证码模板 ID，如 `SMS_123456789` |
| 接入地址 | API 接入地址，默认 `dysmsapi.aliyuncs.com`，通常无需修改 |

::: tip
模板内容中需要包含 `${code}` 变量，系统会自动替换为生成的验证码。例如模板内容：`您的验证码是${code}，5分钟内有效。`
:::

## 腾讯云短信 {#tencent}

| 字段 | 说明 |
|------|------|
| SecretId | 腾讯云 API 密钥 ID |
| SecretKey | 对应的密钥 |
| 短信应用 ID | 短信应用 SDK AppID（SmsSdkAppId） |
| 短信签名 | 已审核通过的短信签名 |
| 模板 ID | 已审核通过的验证码模板 ID |
| 模板参数顺序 | 按模板占位符顺序填写参数名，逗号分隔，如 `code,minutes`（默认 `code`） |
| 国家码 | 号码无 `+` 前缀时自动补全，默认 `+86` |
| 地域 | API 请求地域，默认 `ap-guangzhou` |

## 通用 HTTP {#generic-http}

通用 HTTP 渠道可以对接任意支持 HTTP 接口的短信平台。系统通过参数名映射的方式构建请求，支持签名验证。

| 字段 | 说明 |
|------|------|
| 请求地址 | 短信网关 API 完整地址 |
| 请求方法 | POST 或 GET |
| POST 编码 | POST 请求的编码方式：`form`（表单）或 `json`，仅 POST 生效 |
| 手机号参数名 | 传递手机号的参数名，默认 `phone` |
| 短信内容参数名 | 传递短信内容的参数名，默认 `text` |
| 短信内容模板 | 短信内容模板，`{code}` 会被替换为验证码，如 `您的验证码是 {code}，5分钟内有效` |
| 附加参数 | 每行一个 `key=value`，会作为额外参数一起发送 |
| 请求头 | 自定义请求头，每行一个 `Key: Value` |
| 签名方式 | 支持 `无`、`MD5`、`HMAC-SHA256` 三种签名方式 |
| 签名参数名 | 签名结果放入的参数名，默认 `sign` |
| 签名密钥 | 用于计算签名的密钥 |
| 时间戳参数名 | 如需传递时间戳，填写参数名 |
| 成功标识 | 响应内容包含该字符串时视为发送成功 |

::: tip
通用 HTTP 渠道非常灵活，可以对接大多数国内外短信平台。系统会将手机号和短信内容按照配置的参数名发送到指定地址，您只需在短信平台文档中找到对应的参数名填入即可。
:::

## 短信宝 {#smsbao}

[短信宝](https://www.smsbao.com/)是面向中小站长的低价短信平台，无需企业认证即可使用，作为官方内置插件提供。

| 字段 | 说明 |
|------|------|
| 用户名 | 短信宝平台用户名 |
| ApiKey | 在短信宝后台获取，推荐使用 ApiKey 而非密码 MD5 |
| 短信签名 | 短信内容中的签名，如填写 `Novaix` 则发送时自动添加【Novaix】前缀 |
| 短信模板 | 短信内容模板，`{code}` 会被替换为实际验证码 |

::: tip
短信宝是通过插件系统提供的渠道。安装后需要在「插件管理」中配置参数，然后到「短信服务」设置中将当前渠道切换为短信宝。
:::

## 配合手机号注册使用 {#with-phone-registration}

配置好短信渠道后，用户即可通过手机号注册和登录。手机号注册入口由短信服务是否启用决定，无需额外开关。但用户注册本身仍受「系统设置」→「注册设置」中的「开放注册」开关控制，请确保已开启。详见[用户管理](./user)。
