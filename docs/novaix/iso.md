# ISO 管理 {#iso}

ISO 管理功能允许管理员添加 ISO 镜像文件（如操作系统安装盘、工具光盘等），用户可以将 ISO 挂载到实例的虚拟光驱中使用。

## 管理 ISO {#manage}

### 添加 ISO {#add}

在管理面板的「ISO 管理」页面，点击「添加 ISO」：

1. 填写 ISO 的名称、文件名和描述信息，创建 ISO 记录
2. 将 ISO 文件手动放置到 ISO 存储目录中（默认路径为 `data/images/iso/`，由 `storage.image_dir` 配置项决定），文件名需与创建时填写的文件名一致
3. 在 ISO 列表中将该记录标记为「就绪」状态

::: tip
当前 ISO 不支持浏览器上传或 URL 下载，需要管理员通过 SCP、SFTP 等方式将文件传输到服务器上。如果配置了[对象存储](./storage)，ISO 文件也会异步归档到远程存储。
:::

### 删除 ISO {#delete}

在 ISO 列表中点击「删除」，会同时删除数据库记录和磁盘上的文件。

::: warning
删除 ISO 前请确保没有实例正在使用该 ISO。如果有实例挂载了该 ISO，删除不会影响已挂载的实例，但该 ISO 将无法再被其他实例挂载。
:::

## 用户挂载和卸载 {#mount-unmount}

用户可以在实例详情页中管理 ISO 挂载：

### 挂载 ISO {#mount}

点击实例详情页中的「挂载 ISO」按钮，从可用 ISO 列表中选择需要的 ISO 文件。挂载操作是异步执行的，系统会创建一个任务来完成 ISO 的挂载，您可以在任务列表中查看进度。

挂载 ISO 时会自动设置启动优先级，确保实例优先从 ISO 引导启动。安装完成后卸载 ISO 即可恢复从硬盘启动。

#### 双 ISO 挂载

挂载时可以同时选择一个「驱动 ISO」，系统会将两个 ISO 分别挂载为两个虚拟光驱设备。这在安装 Windows 时特别有用——系统 ISO 用于安装，驱动 ISO 提供 VirtIO 存储和网络驱动。

挂载完成后，实例内部会出现一个或两个虚拟光驱设备，您可以像使用物理光盘一样访问 ISO 中的内容。

### 卸载 ISO {#unmount}

在实例详情页中点击「卸载 ISO」即可。同样是异步操作，完成后虚拟光驱设备会被移除。

::: tip
常见用途：
- 使用 ISO 安装特定版本的操作系统（如 Windows）
- 挂载驱动光盘安装驱动程序（如 VirtIO 驱动）
- 使用救援工具盘修复系统
:::
