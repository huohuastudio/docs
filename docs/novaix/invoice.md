# 发票管理 {#invoice}

Novaix 内置发票管理功能，用户可以根据消费金额申请开具增值税电子普通发票。管理员在外部税务系统完成开票后，将发票号码和电子发票文件录入系统，用户即可查看和下载。

## 启用发票功能 {#enable}

发票功能默认关闭。在管理面板的「系统设置 → 发票」中启用：

| 字段 | 说明 |
|------|------|
| 发票功能 | 开启后，用户端将显示发票入口 |
| 最低开票金额 | 用户申请开票时的最低金额限制，默认 100 元 |
| 开票须知 | 展示在用户申请页面的说明文字，如开票周期、注意事项等 |

启用后，用户端导航会自动显示「发票」入口；关闭后入口自动隐藏，但已有的发票记录仍可查看。

## 可开票额度 {#quota}

用户的可开票额度按以下规则计算：

```
可开票额度 = 净消费金额 - 已开票金额
```

- **净消费金额** = 累计消费（订单支付、附加 IP、流量超额、流量包、按小时扣费、换 IP 费用）- 累计退款
- **已开票金额** = 所有状态为「待审核」「待开票」「已开票」的发票申请金额之和

退款会减少可开票额度。已取消的发票申请会释放对应额度。

## 发票抬头 {#profile}

用户在申请开票前，需要先保存发票抬头。支持两种类型：

| 类型 | 必填字段 |
|------|----------|
| 个人 | 抬头名称、接收邮箱 |
| 企业 | 抬头名称、纳税人识别号、接收邮箱 |

用户可以保存多个抬头，并设置一个默认抬头。每个用户最多只能有一个默认抬头。

## 申请流程 {#flow}

### 用户端

1. 进入「发票」页面，查看可开票额度
2. 选择已保存的发票抬头
3. 填写开票金额（不超过可开票额度，不低于最低开票金额）
4. 提交申请

提交后，发票进入「待审核」状态，申请金额立即从可开票额度中扣减。

### 管理端

1. 在「发票管理」中查看待审核的申请
2. 审核通过后，状态变为「待开票」
3. 在外部税务系统或开票软件中完成开票
4. 回到系统，填写发票号码并上传电子发票文件
5. 标记为「已开票」，用户即可查看和下载

## 发票状态 {#status}

| 状态 | 说明 |
|------|------|
| 待审核 | 用户已提交，等待管理员审核 |
| 待开票 | 审核通过，等待管理员在外部系统开票 |
| 已开票 | 开票完成，用户可下载电子发票 |
| 已驳回 | 审核未通过，用户可查看驳回原因 |

用户可以取消「待审核」状态的申请，额度会立即释放。已审核或已开票的申请不可取消。

## 注意事项 {#notes}

::: warning
- 发票申请提交后即占用额度，即使尚未开票。取消申请后额度会释放
- 管理员驳回申请后，对应额度自动释放，用户可修改信息后重新申请
- 发票抬头在提交申请时会被快照保存，后续修改抬头不影响已提交的申请
- 有进行中的发票申请（待审核或待开票）时，对应的抬头无法删除
- 发票文件支持站内上传路径或 HTTP/HTTPS 外链
:::
