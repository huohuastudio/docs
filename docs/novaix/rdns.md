# 反向 DNS {#rdns}

反向 DNS（rDNS）用于将 IP 地址反向解析为域名。许多邮件服务器和安全策略会检查 IP 的反向解析记录来验证发送方身份，配置正确的 rDNS 有助于提高邮件送达率和服务可信度。

## 配置 DNS 后端 {#configure-backend}

rDNS 功能需要在「系统设置」中配置 DNS 后端后才能使用。目前支持 PowerDNS 作为 DNS 后端：

| 设置项 | 说明 |
|--------|------|
| 启用 rDNS | 开启反向 DNS 功能 |
| API 地址 | PowerDNS HTTP API 地址 |
| API 密钥 | PowerDNS API 认证密钥 |

::: warning
请确保 Novaix 服务器能够访问 PowerDNS 的 API 端口。PowerDNS 需要预先创建好反向解析区域（如 `60.25.103.in-addr.arpa`），Novaix 只负责管理具体的 PTR 记录，不会自动创建区域。
:::

## 管理 rDNS 记录 {#manage}

在管理面板的「反向 DNS」页面，您可以管理所有 IP 的反向解析记录。

### 创建记录 {#create}

点击「添加记录」，选择一个 IP 地址并填写对应的主机名：

| 字段 | 说明 |
|------|------|
| IP 地址 | 从 IP 池中选择一个 IP |
| 主机名 | 反向解析的域名（FQDN 格式，如 `vps1.example.com`） |

::: warning
主机名必须是合法的完全限定域名（FQDN），需要满足：
- 至少包含两段（如 `host.example.com`），不能是 `localhost` 这样的单段名称
- 每段长度 1-63 字符，总长度不超过 253 字符
- 只能包含字母、数字和连字符（`-`），不能以连字符开头或结尾
:::

### 记录状态 {#status}

rDNS 记录有以下几种状态：

| 状态 | 说明 |
|------|------|
| 待同步 | 记录已创建或修改，等待同步到 DNS 服务器 |
| 已同步 | 成功同步到 DNS 服务器 |
| 同步失败 | 同步时发生错误，可查看具体错误信息 |

记录创建或修改后会自动触发同步。如果同步失败，您可以在列表中查看错误信息，修正问题后重试同步。

### 批量同步 {#sync-all}

点击「批量同步」按钮，可以重新同步所有状态为「待同步」和「同步失败」的记录。适用于 DNS 服务器临时不可用后恢复的场景。

## 实例关联 {#instance-association}

当实例被删除时，该实例 IP 上的 rDNS 记录会被自动清理。如果实例更换了 IP 或移除了附加 IP，旧 IP 的 rDNS 记录**不会自动删除**，需要管理员手动处理。更换 IP 后请为新 IP 手动创建新的 rDNS 记录。

::: tip
如果您的用户主要用实例来搭建邮件服务器，建议在创建实例后及时配置 rDNS 记录，将 IP 解析到用户的邮件域名（如 `mail.example.com`）。这样可以显著提高邮件服务的可用性和送达率。
:::
