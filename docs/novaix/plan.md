# 套餐管理 {#plan}

套餐定义了用户可以购买的实例规格和价格。您可以创建不同配置、不同价位的套餐来满足各类用户的需求。

在创建套餐之前，请确保已完成以下准备工作：
1. 至少添加并初始化了一个[节点](./node)
2. 创建了与节点网络名称匹配的 [IP 池](./ip-pool)并有可用 IP
3. 至少导入了一个[镜像](./image)并已分发到该节点

## 创建套餐 {#create-plan}

在管理面板的「套餐管理」页面创建套餐，需要配置以下信息：

### 基本信息 {#basic-info}

| 字段 | 说明 |
|------|------|
| 名称 | 套餐显示名称，如「入门型 1C1G」 |
| 所属分组 | 套餐分组，用于在用户购买页面中分类展示 |
| 类型 | 虚拟机（VM）或 容器（Container） |
| 架构 | CPU 架构，`amd64` 或 `arm64` |
| 可用节点 | 该套餐可使用的节点，实例将在选定的节点上创建 |
| 排序 | 展示顺序，数字越小排越前 |

### 资源配置 {#resources}

| 字段 | 说明 |
|------|------|
| CPU | CPU 核心数 |
| 内存 | 内存大小（MB） |
| 磁盘 | 磁盘空间（GB） |
| 带宽 | 网络带宽限制 |
| 流量 | 月度流量配额（GB），0 表示不限制 |
| IP 数量 | 每个实例分配的公网 IP 数量，默认 1 |
| CPU 限制 | CPU 使用率上限（%），0 表示不限制 |
| Profile 名称 | 节点上使用的 Profile 名称（可选，高级配置） |
| 存储池 | 节点上使用的存储池名称（可选） |
| 网络名称 | 节点上使用的网络名称（可选） |

::: tip
用户下单和支付时，系统会自动检查目标节点的 CPU、内存、磁盘余量以及 IP/NAT 端口可用性。资源不足时会拦截下单或支付，避免付款后实例创建失败。但资源预检是 best-effort 的快照检查，不做资源预留，高并发场景下仍有极小概率出现资源竞争。建议根据节点实际配置合理规划套餐规格和库存。
:::

### 定价 {#pricing}

Novaix 支持多周期定价，您可以为每个计费周期设置不同的价格：

| 周期 | 说明 |
|------|------|
| 按小时 | 按使用时长实时从余额扣费，适合临时或弹性需求 |
| 月付 | 按月计费价格 |
| 季付 | 按季度计费价格 |
| 年付 | 按年计费价格 |

不设置价格的周期将不会展示给用户。例如，如果您只希望提供月付和年付，只需设置这两个周期的价格即可。按小时计费的详细说明请参考[订单与计费](./order#hourly-billing)。

::: tip
长周期通常会给予一定折扣以吸引用户长期付费。例如月付 30 元的套餐，季付可设为 80 元（约 8.9 折），年付设为 280 元（约 7.8 折）。
:::

### NAT 模式 {#nat-mode}

启用 NAT 模式后，该套餐创建的实例将共享公网 IP，通过端口转发与外界通信，适合低成本 VPS 产品。

| 字段 | 说明 |
|------|------|
| NAT 模式 | 开启后实例共享公网 IP |
| 端口数量 | 每个实例分配的端口数量，默认 20 |
| 独享 IP 月价 | 用户购买独享公网 IP 的月费用，需大于 0 才启用独享 IP 功能 |
| 最大独享 IP 数 | 需设置大于 0 作为启用标志（NAT 实例实际上限固定为 1） |

NAT 模式下的变化：
- 实例仍分配独立内网 IP（IP 数量固定为 1），但不分配独立公网 IP
- 购买后自动从节点的[共享 IP](./shared-ip) 中分配一段连续端口
- 第一个端口自动映射到 SSH（端口 22），其余端口内外一致可自由使用
- 支持购买[独享 IP](./shared-ip#dedicated-mode)，获得独立出口和全端口入站

::: warning
使用 NAT 模式前，请确保已在目标节点上创建了[共享 IP](./shared-ip)，且有足够的可用端口。如需支持独享 IP 功能，还需在节点上添加「独享」模式的 IP。
:::

### 库存 {#stock}

您可以设置套餐的库存数量。当库存为 0 时，用户将无法购买该套餐。设置为 -1 表示不限库存。

::: tip
即使库存充足，如果对应网络的 [IP 池](./ip-pool)没有空闲 IP，用户同样无法购买。库存和 IP 是两个独立的限制条件，两者都需要满足才能成功创建实例。
:::

## 套餐分组 {#plan-groups}

套餐分组用于在用户购买页面中对套餐进行分类展示。例如您可以创建「香港节点」「美国节点」等分组，将不同地区的套餐归类。

分组名称会直接展示给用户，建议使用用户能理解的名称（如地区或用途），而不是内部编号。

## 可用镜像 {#available-images}

创建套餐时，您需要选择该套餐可使用的镜像。用户购买该套餐的实例时，只能选择这些镜像作为操作系统。

::: warning
请确保选择的镜像已经分发到套餐所关联的节点上。如果镜像只存在于 Novaix 的镜像管理中，但尚未分发到目标节点，用户选择该镜像创建实例时会失败。
:::

## 试建实例 {#trial}

套餐创建后，可以在套餐列表的操作菜单中点击「试建实例」，选择一个节点和镜像，系统会用该套餐的配置在节点上创建一个临时实例，验证完整的创建→启动链路是否正常，完成后自动销毁。

试建任务会在任务面板中显示实时日志和各步骤耗时，方便排查配置问题。

::: tip
- 试建不分配公网 IP 和 NAT 端口，验证的是实例创建和启动链路，不包含网络交付。IP/NAT 的可用性由下单时的资源预检保障。
- 下架状态的套餐也可以试建，方便在上架前验证配置是否正确。
- 同一节点上同时只允许一个试建任务，防止占用过多资源。
:::

## 修改和删除套餐 {#edit-delete}

- **修改套餐**：修改套餐的配置或价格不会影响已创建的实例，只影响后续新购和续费订单
- **删除套餐**：如果套餐下还有关联的实例，系统会拒绝删除。建议先将套餐下架（库存设为 0）而不是直接删除，待所有实例处理完毕后再执行删除
