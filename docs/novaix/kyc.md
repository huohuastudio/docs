# 实名认证 {#kyc}

Novaix 支持用户实名认证（KYC），管理员可以要求用户在购买实例前完成身份核验。实名认证采用插件化设计，支持多种核验渠道和认证模式。

## 认证模式 {#modes}

| 模式 | 说明 |
|------|------|
| 二要素验证 | 仅校验姓名与身份证号是否一致，用户提交后即时返回结果 |
| 人脸识别 | 用户提交姓名和身份证号后，跳转至第三方 H5 页面完成活体检测，验证通过后自动返回 |

管理员可在「系统设置」→「实名认证」中切换认证模式。

## 支持的渠道 {#channels}

| 渠道 | 二要素验证 | 人脸识别 | 说明 |
|------|:---:|:---:|------|
| 阿里云身份核验 | ✓ | ✓ | 需开通阿里云实人认证服务 |
| 腾讯云身份核验 | ✓ | ✓ | 人脸识别需额外配置 RuleId（在腾讯云人脸核身控制台创建业务流程后获取） |

## 配置 {#configuration}

在管理面板的「系统设置」→「实名认证」中：

1. 启用实名认证
2. 选择核验渠道并填写 API 密钥
3. 选择认证模式（二要素验证 或 人脸识别）

### 腾讯云人脸识别配置 {#tencent-face}

使用腾讯云人脸识别模式时，除了 SecretId 和 SecretKey 外，还需要配置 **RuleId**：

1. 登录 [腾讯云人脸核身控制台](https://console.cloud.tencent.com/faceid)
2. 进入「自助接入」，创建业务流程
3. 审核通过后获取 RuleId，填入设置中

### 阿里云人脸识别配置 {#aliyun-face}

阿里云渠道使用同一组 AccessKey，无需额外配置。确保已在阿里云控制台开通实人认证服务即可。

## 认证流程 {#verification-flow}

### 二要素模式 {#two-factor-flow}

1. 用户在「个人资料」中填写真实姓名和身份证号
2. 系统调用核验渠道进行身份核验
3. 核验通过后，认证状态更新为「已认证」

### 人脸识别模式 {#face-flow}

1. 用户在「个人资料」中填写真实姓名和身份证号，点击「开始人脸识别」
2. 页面跳转至第三方 H5 页面，用户完成活体检测
3. 检测完成后自动跳转回系统，系统查询核验结果
4. 核验通过后，认证状态更新为「已认证」

## 管理 {#management}

管理员可以在「用户管理」中查看每个用户的实名认证状态。如果您希望强制要求用户实名认证才能购买实例，可以在「系统设置」中开启实名认证要求。

::: warning
实名认证服务由第三方云平台提供，每次核验会产生费用。人脸识别模式的费用通常高于二要素验证，请在对应云平台的控制台中查看计费详情。
:::

::: tip
实名认证不是必须的。如果您的业务不要求用户实名，可以不配置此功能，用户将可以直接购买实例。
:::
