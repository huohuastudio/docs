# 流量包 {#traffic-package}

流量包为实例提供额外的流量配额，适用于流量不够用的用户按需购买。Novaix 支持两种流量包模式，管理员可以灵活配置流量包产品，用户在实例详情页自助购买。

## 流量包类型 {#types}

| 类型 | 说明 |
|------|------|
| 叠加包 | 在现有流量配额上叠加额外流量，一次性生效 |
| 重置包 | 清零当月已使用流量和计费标记，相当于重新开始当月计量，不改变套餐的月度流量上限 |

::: tip
叠加包适合偶尔突发使用的场景，重置包适合月初流量已用尽但不需要更多配额的用户。两种类型可以同时创建，用户按需选择。
:::

## 创建流量包 {#create}

在管理面板的「流量包管理」中创建流量包，需要配置以下信息：

| 字段 | 说明 |
|------|------|
| 名称 | 流量包显示名称，如「50GB 叠加包」 |
| 类型 | 叠加 或 重置 |
| 流量 | 流量大小（GB），仅叠加包必填；重置包不使用此字段（可填 0） |
| 价格 | 购买价格（必须大于 0） |
| 适用套餐 | 限制哪些套餐的实例可以购买此流量包，留空表示所有套餐均可购买 |
| 排序 | 展示顺序，数字越小排越前 |
| 启用 | 是否上架 |

::: warning
适用套餐是一个重要的限制条件。如果您为不同地区的节点提供不同价格的流量包，可以通过适用套餐来确保用户只能购买对应地区的流量包，避免低价流量包被用于高成本节点的实例。
:::

## 用户购买 {#purchase}

用户可以在实例详情页的「流量」标签中查看当前流量使用情况，并购买可用的流量包。购买后会生成一笔订单，支付完成后流量即时到账。

::: warning
只有套餐设置了流量限额（`traffic > 0`）的实例才能购买流量包。不限流量的实例不会展示流量包购买入口。
:::

## 月度流量重置 {#monthly-reset}

系统每月 1 日自动重置实例的月度流量计数。如果上月实际用量超出了套餐月度配额，超出部分会先从叠加包余量中扣除，剩余的叠加包流量保留到下月继续使用。

::: warning
流量重置由定时任务在每月 1 日执行。如果实例是在月中创建的，首月的流量配额仍为完整的月度额度，不会按天折算。
:::
